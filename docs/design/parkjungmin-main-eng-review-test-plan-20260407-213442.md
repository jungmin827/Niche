## Affected Routes
- `POST /v1/interests` — 관심사 생성
- `GET /v1/me/interests` — 내 관심사 목록 (depth_score 포함)
- `GET /v1/interests/{id}` — 관심사 상세
- `POST /v1/interests/{id}/logs` — 로그 추가
- `PATCH /v1/interests/{id}/logs/{logId}` — 로그 수정
- `DELETE /v1/interests/{id}` — 관심사 삭제 (cascade)

## Key Interactions to Verify
- 관심사 생성 → 로그 추가 → depth_score가 양수로 바뀌는지 확인
- 로그 0개일 때 depth_score가 None/null로 반환되는지 확인
- Interest 삭제 시 연결된 모든 Log도 soft-delete 되는지 확인
- 타인의 interest_id로 로그 추가 시 404 반환 확인
- 타인의 interest_id로 Interest 삭제 시 404 반환 확인

## Edge Cases
- `startedAt`이 미래 날짜 → 422 반환
- `text`가 빈 문자열 또는 공백만 → 422 반환
- `text`가 2000자 초과 → 422 반환
- 유효하지 않은 `tag` 값 → 422 반환
- 관심사 0개인 유저 → 빈 배열 반환

## Critical Paths
- 전체 MVP 흐름: 관심사 추가 → 기록 3개 추가 → depth_score 확인 → 삭제 (cascade 검증)
