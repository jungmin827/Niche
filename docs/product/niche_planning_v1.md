## 1. 서비스 개요

### 서비스명
**NichE**

### 서비스 한 줄 정의
NichE는 책, 취향, 관찰, 사유 같은 **깊고 마이너한 관심사에 몰입한 시간**을 기록하고, 증명하고, 감각적으로 공유하게 만드는 앱 서비스이다.

### 핵심 출발점
NichE의 아이디어는 **Nike Run Club가 러닝 문화를 바꾼 방식**에서 출발한다.

Nike Run Club는 단순히 달리기 기록을 저장하는 앱이 아니라,
- 몇 km를 뛰었는지 인증하게 만들고,
- 그것을 스토리에 공유하게 만들고,
- 약간의 허영심과 자기표현 욕구를 건강한 행동의 지속 동력으로 바꾸었다.

NichE는 이 구조를 러닝이 아니라 다음 영역에 적용하고자 한다.
- 책 읽기
- 취향 탐구
- 마이너한 관심사 딥다이브
- 일상 속 관찰과 사유

즉, NichE는 **깊은 취향 생활을 더 매력적으로 기록하고 자랑할 수 있게 만들어**, 사람들이 자극적인 콘텐츠 소비 대신 자신만의 취향 세계에 시간을 쓰도록 유도하는 서비스이다.

## 2. 문제 정의

현재 많은 사람들은 다음과 같은 상태에 놓여 있다고 본다.
- 쇼트폼 중심의 빠른 콘텐츠 소비에 피로감을 느낀다.
- 깊은 몰입, 사유, 독서, 취향 탐구 같은 활동을 하고 싶어 한다.
- 하지만 이러한 활동은 외부적으로 드러나기 어렵고, 지속 동기를 만들기 어렵다.
- 러닝처럼 기록과 인증을 통해 동기를 강화하는 문화가 아직 충분히 형성되어 있지 않다.

NichE는 여기서 다음 문제를 해결하고자 한다.

### 해결하려는 핵심 문제
1. 깊은 취향 생활은 기록과 지속의 동력이 약하다.
2. 취향 탐구는 외부적으로 보여주기 어렵다.
3. 독서, 관찰, 사유 같은 활동은 성취감 구조가 약하다.
4. 마이너한 취향은 혼자만의 경험으로 끝나기 쉽다.

### NichE의 해결 방식
- 몰입 시간을 세션 단위로 기록하게 한다.
- 기록된 시간을 감각적인 템플릿으로 외부에 공유할 수 있게 한다.
- AI 기반 문제/회고를 통해 "내가 진짜 이해하고 축적했는가"를 확인하게 한다.
- 점수, 칭호, 아카이브를 통해 축적의 감각을 만든다.
- 피드(Text Wave)를 통해 지금 다른 사람들이 무엇을 탐구하는지 감각적으로 흘려보게 한다.

## 3. 서비스 철학

NichE는 단순한 생산성 앱이 아니다.
NichE는 단순한 독서 기록 앱도 아니다.
NichE는 일반적인 SNS와도 다르다.

NichE의 본질은 다음 문장으로 정리된다.

> **NichE는 취향을 소비하는 앱이 아니라, 취향을 축적하고 증명하는 앱이다.**

### NichE의 핵심 정서
- 조용한 몰입
- 깊이 있는 취향
- 감각적인 기록
- 축적의 미학
- 사유와 관찰의 습관
- 자랑이 가능한 취향 생활

### NichE가 지향하는 문화
- 책을 읽는 시간이 멋있어지는 문화
- 취향을 깊게 파는 것이 자랑이 되는 문화
- 일상의 관찰과 사유를 글로 남기는 문화
- 마이너한 관심사가 오히려 매력적으로 보이는 문화

## 4. 핵심 사용자 행동

NichE의 핵심 행동은 **세션 기반 딥다이브**이다.

### 기본 흐름
1. 사용자는 특정 관심사/책/주제에 대해 몰입할 시간을 가진다.
2. 기본 세션 단위는 **15분**이다.
3. 세션이 끝나면 사용자는 자신이 무엇을 보았고, 무엇을 공부했고, 어떤 내용을 다뤘는지 **짧게 기록**한다.
4. 여러 세션이 쌓이면 서비스는 그 기록을 기반으로 사용자의 탐구 히스토리를 축적한다.
5. AI는 이 기록을 바탕으로 사용자의 이해와 회고를 유도하는 질문/문제를 생성한다.
6. 사용자는 답변과 복습을 통해 점수와 랭크를 쌓는다.
7. 그 결과는 감각적인 템플릿으로 외부 SNS에 공유되거나, 서비스 내부 아카이브에 축적된다.

### 세션이 중요한 이유
세션은 NichE에서 단순 타이머 기능이 아니다.
세션은 사용자가 자신의 취향에 시간을 썼다는 사실을 증명하는 **행동 단위**이며,
이 행동이 곧 서비스의 중심 데이터가 된다.

## 5. 차별점

NichE의 차별점은 단순한 기록이 아니라 **깊이에 대한 증명 구조**에 있다.

### 일반 독서/기록 앱과의 차이
일반 기록 앱:
- 읽었다
- 봤다
- 저장했다
- 메모했다

NichE:
- 실제로 시간을 썼다
- 그 시간을 기록했다
- 그 기록을 바탕으로 회고했다
- AI 문제를 통해 이해와 몰입을 다시 확인했다
- 점수/칭호/아카이브로 축적했다
- 감각적인 템플릿으로 자랑하고 공유할 수 있다

### 핵심 차별점 요약
1. **세션 기반 몰입 기록**
2. **AI 기반 회고/문제 생성**
3. **점수/칭호를 통한 깊이의 가시화**
4. **외부 공유 가능한 감각적 템플릿**
5. **블로그와 하이라이트가 공존하는 아카이브 구조**
6. **Text Wave Feed — 타인의 탐구를 감각적으로 흘려보는 공간**

## 6. 정보 구조 — 4탭 구조 (현재 확정안)

> ⚠️ 이전 문서의 3탭 구조(세션 / 피드 / 아카이브)는 **폐기**되었다.
> 블로그가 아카이브에서 독립되어 별도 탭으로 분리되었으며, 피드 개념은 Text Wave로 전면 재설계되었다.

NichE의 현재 탭 구조는 다음 **4개**로 확정한다.

- **세션**
- **피드 (Text Wave)**
- **아카이브**
- **블로그**

### 6.1 세션
서비스의 메인 탭이자 핵심 행동 공간.

역할:
- 딥다이브 세션 시작
- 세션 진행/종료
- 세션 후 간단한 기록 작성
- 세션 누적 정보 확인
- AI 문제 생성의 원천 데이터 축적

세션은 NichE의 중심이며, 모든 구조는 세션 데이터를 기반으로 확장된다.

### 6.2 피드 (Text Wave / Trend Radar)

> ⚠️ 기존 소셜 포스트 수직 스크롤 피드 개념은 **전면 폐기**되었다.
> 피드 탭은 **"Infinite Marquee Text Wave"** 로 완전히 재설계되었다.

#### 개념
지난 24시간 공개 하이라이트 제목들이 화면을 가로지르며 수평 흐름으로 보여지는 공간.
SNS 피드가 아닌 **포에틱 에디토리얼 티커 테이프** 또는 **현대 아트 설치물** 같은 느낌이어야 한다.

지금 이 순간 다른 사람들이 무엇을 탐구하고 있는지를 조용하고 감각적으로 흘려보여준다.
사용자는 스크롤하지 않는다 — 텍스트가 스스로 흐른다.

#### 시각 구조: 3개의 패럴랙스 레이어
| 레이어 | 폰트 크기 | 색상 | 속도 | 방향 |
|---|---|---|---|---|
| Layer 1 (Background) | 매우 큰 | `#EFEFEA` (very light gray) | 매우 느림 | Left → Right |
| Layer 2 (Middle) | 중간 | `#555555` (dark gray) | 중간 | Right → Left |
| Layer 3 (Foreground) | 기본 | `#111111` (deep black), bold | 빠름 | Left → Right |

#### 인터랙션
- 각 텍스트 아이템은 `Pressable`로 감싼다.
- `onPressIn`: 모든 레이어 애니메이션을 부드럽게 감속 → 일시정지
- `onPress`: 해당 하이라이트의 9:16 템플릿 이미지를 BottomSheet Modal로 오픈
- `onRelease` / Modal 닫기: 정상 속도로 부드럽게 재가속

#### 데이터 원천
- API: `GET /v1/feed/wave`
- 조건: `visibility = 'public'` AND `created_at >= NOW() - 24h` AND `deleted_at IS NULL`
- 최대 30~50개, 매 요청마다 랜덤 순서 반환 (`ORDER BY RANDOM()`)
- 커서 페이지네이션 없음

#### 피드 탭이 하지 않는 것
- 댓글 작성 / 좋아요 / 반응 (MVP 제외)
- 수직 스크롤 카드 리스트
- 팔로우 기반 개인화 피드
- 실시간 핫 콘텐츠 / 인기 급상승 표시

### 6.3 아카이브
개인의 축적과 세계관이 쌓이는 공간.

아카이브는 단순 저장소가 아니라,
**나의 세션 하이라이트가 쌓이는 개인 공간**이다.

> 참고: 블로그 기록(글쓰기)은 별도 탭으로 분리되었다.

아카이브 내부:
- 세션 하이라이트 — 깊게 탐구한 취향 세션 기록. 인스타그램 스토리 하이라이트처럼 축적.
- 프로필 통계 — 총 세션 수, 누적 몰입 시간, 현재 랭크/칭호

### 6.4 블로그 (Dump)

> 참고: 블로그는 아카이브에서 독립하여 별도 탭이 되었다.

블로그 탭은 전체 유저 글 탐색 공간이 **아니다**.
사용자 자신의 깊은 생각을 장문으로 기록하는 **개인 Dump 전용 공간**이다.

역할:
- 일상의 관찰, 사물과 현상에 대한 해석
- 감정, 기분, 사유
- 취향과 연결된 생각의 글
- 세션 이후 떠오르는 더 깊은 생각의 확장

사용자의 마이페이지가 곧 하나의 개인 블로그가 되는 구조를 지향한다.

## 7. MVP 핵심 기능

NichE의 MVP는 모든 기능을 한 번에 구현하는 것이 아니라,
서비스의 가장 강력한 핵심 경험을 먼저 성립시키는 데 집중해야 한다.

### MVP에서 가장 중요한 핵심
**감각적인 템플릿을 통한 기록 공유 경험**

### MVP 핵심 기능 목록
1. 15분 세션 시작/종료
2. 세션 종료 후 짧은 기록 입력
3. 세션 기록 누적
4. AI 기반 간단한 회고/문제 생성
5. 결과 기반 점수/칭호 부여
6. 외부 SNS 공유용 템플릿 생성
7. 아카이브 내 세션 하이라이트 저장
8. **블로그 탭 내 장문 글 작성/조회**
9. **피드 탭: Text Wave (Trend Radar) — 24h 공개 하이라이트 마키 표시**

### MVP 우선순위 정리
#### 최우선
- 세션
- 기록
- 공유 템플릿
- 아카이브 저장

#### 그 다음
- AI 문제/회고
- 점수/칭호
- Text Wave Feed

#### 이후 확장
- 피드 상호작용 고도화 (댓글, 반응)
- 취향 기반 추천
- 커뮤니티 기능

## 8. AI 기능의 역할

NichE에서 AI는 전면에 나서는 주인공이 아니다.
AI는 사용자의 취향 생활을 대신하는 것이 아니라,
사용자가 직접 쌓은 기록을 더 의미 있게 만들어주는 보조 장치여야 한다.

### AI의 핵심 역할
1. 세션 기록 기반 질문/문제 생성
2. 복습과 회고 유도
3. 사용자의 이해도/몰입도에 대한 게임화 보조
4. 템플릿용 짧은 카피 보조 생성 가능

### AI 문제의 의미
예를 들어 사용자가 1시간 동안 4개의 세션을 통해 어떤 분야를 딥다이브했다면,
그 4개의 기록을 기반으로 AI가 주관식 질문을 던진다.

이 질문은 단순 퀴즈가 아니라:
- 내가 무엇을 봤는지 상기시키고
- 어떤 관점을 얻었는지 정리하게 하며
- 몰입한 시간을 다시 내 것으로 만드는 장치가 된다.

즉, AI는 NichE에서 **깊이를 점검하고 기억을 강화하는 도구**이다.

## 10. 디자인 / 브랜딩 무드

NichE의 디자인은 화려한 장식성보다,
**정제된 감각과 타이포 중심의 무드**를 지향한다.

### 핵심 디자인 원칙
- **Helvetica 기반 타이포그래피**
- **White / Black 중심 모노크롬 무드**
  - 배경: Pure White (`#FFFFFF`) 또는 Deep Black (`#111111`)
  - 텍스트 계층: `#111111` / `#555555` / `#EFEFEA`
  - **규칙: 밝은 액센트 컬러, 그라디언트, 과한 드롭섀도 금지**

## 11. 블로그 도입의 의미

NichE에는 단순한 피드형 SNS가 아니라,
사용자의 생각을 오래 남길 수 있는 **블로그적 구조**가 중요하다.

### 블로그가 필요한 이유
사용자는 단지 세션만 기록하는 것이 아니라,
일상에서 느끼는 다음과 같은 생각들을 기록할 수 있어야 한다.
- 어떤 사물이나 현상에 대한 관찰
- 그때의 기분과 감정
- 사유와 해석
- 취향과 연결된 개인적인 이야기

### 블로그와 세션의 관계
- 세션은 "시간을 쓴 행동"의 기록
- 블로그는 "생각과 세계관"의 기록

즉, NichE는 행동과 사유를 함께 축적하는 구조를 가진다.

## 12. 확장 가능성

NichE는 MVP 이후 다음 방향으로 확장 가능하다.

### 12.1 피드 고도화
- Text Wave에 인터랙션 요소 추가 (반응, 저장)
- 유사 관심사 사용자 연결
- 책/콘텐츠/인사이트 추천
- 취향 기반 팔로우 구조

### 12.2 커뮤니티화
- 특정 관심사별 공간
- 댓글/대화/반응
- 사용자 간 취향 교류

### 12.3 컬렉션 요소 강화
- 특정 책/주제/장르를 실제로 읽고 탐구한 기록을 기반으로 컬렉션 구축
- "완료"가 아니라 "실제로 읽고 이해했다"는 감각의 시각화

### 12.4 외부 SNS 익스포트 다양화
- 인스타 스토리 템플릿
- 기타 SNS 포맷에 맞는 내보내기
- 브랜드 일관성을 유지한 카드/이미지 구조

## 16. 문서 요약

NichE는
- 쇼트폼 중심 문화의 반대편에서,
- 책/취향/관찰/사유를 더 매력적으로 기록하게 만들고,
- 깊은 몰입을 세션 단위로 축적하게 하며,
- AI 기반 회고와 감각적 템플릿을 통해
- 취향 생활을 자랑 가능하고 지속 가능한 문화로 바꾸려는 앱 서비스이다.

현재 NichE의 핵심 구조는 다음과 같다.
- **세션**: 몰입 행동의 시작점
- **피드 (Text Wave)**: 24시간 공개 하이라이트가 흐르는 감각적 발견 공간
- **아카이브**: 세션 하이라이트 기반 개인 세계관 축적 공간
- **블로그**: 생각과 사유를 장문으로 남기는 개인 덤프 공간

이 문서는 NichE의 현행 기준 기획서이며,
이후 모든 세부 문서는 본 문서를 기준으로 파생된다.
