## 문서 목적
이 문서는 **NichE 서비스의 현재 기획 내용을 구조적으로 정리한 기준 문서**이다.
추후 이 문서는 AI 툴, 개발 문서의 입력 컨텍스트로 활용되는 것을 전제로 작성한다.

---

## 1. 서비스 개요

### 서비스 한 줄 정의
NichE는 깊은 관심사(자연 와인, 독립 영화 등)에 몰두한 시간과 기록을 바탕으로 **얼마나 오래, 얼마나 깊이 파고들었는지 증명(Depth Score)하고, 이를 감각적인 카드로 공유하게 만드는 '깊이 증명 엔진(Proof Engine)'**이다.

### 핵심 접근
현존하는 기록 도구들(블로그, 인스타, 노션, Letterboxd)은 "얼마나 깊이" 특정 관심사에 매몰되어 있는지를 한눈에 보여주거나 감각적으로 자랑하게 만들어주지 못한다. NichE는 이를 겨냥하여 **기록의 양과 시간에 비례해 Depth Score가 오르는 형태**로 동기를 부여한다.

---

## 2. 해결하려는 핵심 문제
1. 깊은 취향 생활(독서, 와인, 영화 탐구 등)은 인스타그램에 지속적으로 올리기는 부담스럽지만, 은근슬쩍 깊이를 자랑하고 싶은 지적 허영심이 존재한다.
2. 기존 도구들(노션, 옵시디언)은 완전히 사적이고 폐쇄적이다.
3. 깊이의 정도를 시각적 지표로 증명해주는 버티컬 플랫폼이 부재하다.

---

## 3. 핵심 엔티티 및 스펙 (Proof Engine)

### 3.1 Interest (관심사)
- 사용자는 자신이 파고드는 '관심사'를 하나 이상 생성한다 (예: "자연 와인").
- 시작 날짜(`started_at`)를 설정하여 언제부터 빠져들었는지 명시한다 (과거 날짜 허용 확정).
- 기본적으로 외부에 증명하기 위함이므로 `is_public=true`를 기본으로 한다.

### 3.2 Log (기록)
- 특정 Interest에 대한 단건 기록 (방문, 독서, 테이스팅 노트 등).
- 각 Log는 개별적으로 관리되며, `is_public=false`가 기본값이다. (기록은 사적, 전체 점수/카드는 공개 지향)
- 최소 1자, 최대 2000자의 텍스트 가능. 태그 지정.

### 3.3 Depth Score 계산 로직
- `depth_score = log10(days_since_start + 1) × log10(record_count + 2)`
- 위 공식을 기반으로 관심사의 깊이 수준을 가시화한다. 백엔드에서 실시간 계산하여 소수점 1자리까지 제공한다.
- 향후 (v2) 사진 첨부, 텍스트 길이 등 정성적 요소를 합산 가능성 있음.

### 3.4 Share Card (공유 카드)
- 540x960px 인스타그램 스토리 최적화 세로형 카드.
- 검은 바탕(#1a1a1a), 텍스트 중심의 감각적 무드.
- 내 관심사 이름, Depth Score, 시작일, 기록 개수, 특정 기록 발췌명 포함.
- MVP에서는 `react-native-view-shot`을 활용하여 클라이언트 단에서 캡처 형태로 생성한다.

### 3.5 보존 기능: Session & Zitter
이번 개편에서도 다음 두 컴포넌트는 반드시 살아남는다.
- **Session (세션)**: 15분 단위 등 타이머 기반으로 시간/몰입을 기록하는 기능. Interest 로그 생성과 병목되거나 보완하는 관계로 유지.
- **Zitter (챗봇/AI)**: 단순 GPT 기반의 질의응답 및 상호작용 기능.

## 4. UI 탭 구조 개편

이전의 4탭(Session / Feed / Archive / Blog) 구조는 **전면 폐기**하고 다음의 구조를 따른다.

### 화면 구조 (예상)
1. **Interests (홈)**: 내 Interest 목록 렌더링. 각 카드에 `depth_score`와 타임라인 노출.
2. **Session**: 세션 진행 타이머.
3. **Zitter**: Zitter AI와의 챗봇/대화 공간.

또한 내부 화면으로는:
- **Interest Detail**: 특정 Interest의 전체 Log 리스트 시각화, 로그 추가, 공유 카드 생성 플로우 진입.
